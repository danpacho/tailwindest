## Briefly

`CreateCompiledTailwindest` creates the Tailwindest style object type for
compiler-oriented projects.

Use it with [`@tailwindest/compiler`](/compiler) when nested variant prefixes
should live in object keys, not in leaf class values. This improves authoring
DX because values autocomplete as raw Tailwind utilities while the compiler
derives `dark:`, `hover:`, responsive, data, aria, and arbitrary variant
prefixes from the object path.

## Why It Exists

Normal [`CreateTailwindest`](/apis/createtailwindest) is designed for runtime
Tailwindest authoring. In runtime mode, nested values include their accumulated
variant prefix:

```ts
const runtimeStyle = tw.style({
    dark: {
        hover: {
            backgroundColor: "dark:hover:bg-red-950",
        },
    },
})
```

Compiler-oriented authoring should not require those prefixes in values:

```ts
const compiledStyle = tw.style({
    dark: {
        hover: {
            backgroundColor: "bg-red-950",
        },
    },
})
```

`CreateCompiledTailwindest` types the second shape.

## 1. Type Definition

```ts
import type { CreateCompiledTailwindest } from "tailwindest"
import type { Tailwind, TailwindNestGroups } from "./tailwind"

export type Tailwindest = CreateCompiledTailwindest<{
    tailwind: Tailwind
    tailwindNestGroups: TailwindNestGroups
    groupPrefix?: string
    useArbitrary?: boolean
    useArbitraryVariant?: boolean
}>
```

## 2. Spec

### Generic Parameter: `Config`

`CreateCompiledTailwindest` takes a single generic `Config` object.

- **`tailwind`** (required): The `Tailwind` type generated by
  [`create-tailwind-type`](/create-tailwind-type).
- **`tailwindNestGroups`** (required): The generated variant key union, such as
  `"hover"`, `"focus"`, `"md"`, and `"dark"`.
- **`groupPrefix`** (optional): A string literal prefix for variant keys. For
  example, `groupPrefix: "$"` lets you write `$hover`.
- **`useArbitrary`** (optional, `boolean`): Allows arbitrary string values when
  set to `true`. Defaults to `false`.
- **`useArbitraryVariant`** (optional, `boolean`): Allows arbitrary variant
  object keys such as `data-[state=open]`, `aria-[expanded=true]`, and
  `[&_svg]`. Defaults to `false`.

## 3. Typical Setup

```ts filename="tw.ts"
import {
    createTools,
    type CreateCompiledTailwindest,
    type CreateTailwindLiteral,
} from "tailwindest"
import type { Tailwind, TailwindNestGroups } from "./tailwind"

export type Tailwindest = CreateCompiledTailwindest<{
    tailwind: Tailwind
    tailwindNestGroups: TailwindNestGroups
    useArbitrary: true
    useArbitraryVariant: true
}>

export type TailwindLiteral = CreateTailwindLiteral<Tailwind>

export const tw = createTools<{
    tailwindest: Tailwindest
    tailwindLiteral: TailwindLiteral
    useArbitrary: true
    useTypedClassLiteral: true
}>()
```

## 4. Nested Variant Authoring

`CreateCompiledTailwindest` keeps variant prefixes in object keys:

```ts
const box: Tailwindest = {
    backgroundColor: "bg-red-50",
    dark: {
        backgroundColor: "bg-red-900",
        hover: {
            backgroundColor: "bg-red-950",
        },
    },
}
```

The compiler normalizes this as:

```ts
"bg-red-50 dark:bg-red-900 dark:hover:bg-red-950"
```

The leaf value is intentionally `"bg-red-950"`, not
`"dark:hover:bg-red-950"`.

## 5. Arbitrary Variants

Enable arbitrary variant keys with `useArbitraryVariant: true`:

```ts
export type Tailwindest = CreateCompiledTailwindest<{
    tailwind: Tailwind
    tailwindNestGroups: TailwindNestGroups
    useArbitrary: true
    useArbitraryVariant: true
}>
```

Then use arbitrary variant object keys with raw leaf values:

```ts
const item: Tailwindest = {
    "data-[state=open]": {
        backgroundColor: "bg-blue-500",
        hover: {
            backgroundColor: "bg-blue-600",
        },
    },
    "[&_svg]": {
        width: "w-4",
        height: "h-4",
    },
}
```

## 6. Group Prefix

If your generated variant keys are easier to distinguish with a prefix, set
`groupPrefix`:

```ts
export type Tailwindest = CreateCompiledTailwindest<{
    tailwind: Tailwind
    tailwindNestGroups: TailwindNestGroups
    groupPrefix: "$"
}>
```

Then write prefixed variant keys:

```ts
const card: Tailwindest = {
    backgroundColor: "bg-white",
    $hover: {
        backgroundColor: "bg-slate-50",
    },
}
```

The prefix applies to object keys only. Leaf values stay raw utilities.

## 7. When Not To Use It

Do not use `CreateCompiledTailwindest` for a project that only uses Tailwindest
at runtime without `@tailwindest/compiler`.

For runtime-only projects, use [`CreateTailwindest`](/apis/createtailwindest):

```ts
import type { CreateTailwindest } from "tailwindest"
```

The runtime type expects prefixed nested leaves and matches normal Tailwindest
runtime authoring.

## 8. CSS Transformer Integration

When using [`@tailwindest/css-transformer`](/css-transformer) to migrate source
for compiler projects, use compiled output mode:

```bash
npx @tailwindest/css-transformer --mode compiled
```

That makes transformed nested leaves compatible with
`CreateCompiledTailwindest`:

```ts
{
    dark: {
        hover: {
            backgroundColor: "bg-red-950",
        },
    },
}
```

Use `--mode runtime` when your target type is `CreateTailwindest`.
